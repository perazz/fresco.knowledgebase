Discrete Particle Ignition Kernel (DPIK)

Model input#

The Discrete Particle Ignition Kernel (DPIK) model is implemented based on the original model by Fan and Reitz and Tan and Reitz. In the Matlab structure, kernel model inputs are found under the flame.sparkPlug structure:

electrodes: [] % 1 or 2 electrode locations
nChannelPts: 1 % number of spark channel points
kernel_r0: 0.1000 % initial radius of each kernel parcel
eta_eff: 0.3000 % thermal discharge efficiency [0 to 1]
Qspark: 20 % Electrical spark plug power [W]
strikes: [] % Spark strike events
toFlame_mode: 'TURBLENGTH' % Kernel->Flame transition method
toFlame_val: 25 % Kernel->Flame transition threshold
cema_U: 'BLANCO' % [CEMA only] Spontaneous Flame Speed method ("BLANCO","SANKARAN")

The same settings are found in the flame model's ASCII input (itapeFlame) as optional inputs:

cema_u BLANCO ! CEMA velocity mode: "Sankaran" use IDT; "Blanco" use ERC Autoignition Index
channelPts 1 ! number of spark-channel particles
electrode -0.300000 0.000000 -0.700000 ! electrode 1 of 1: location
r0 1.00000E-001 ! kernel: initial radius [cm]
eta_eff 3.00000E-001 ! kernel: thermal discharge coefficient [-]
toFlame TURBLENGTH 25.00 ! kernel->flame switch. "TURBLENGTH" (follow CM1), "NCELLS" (follow #)
Qspark 2.00000E+001 ! spark plug electrical power [W]
strike at -14.000000 for 8.000000 ! spark strike 1 of 1

Spark Plug geometry#

Spark plug geometry is represented as either one or two 3D locations, which represent the electrode endpoints:

  • If only one electrode is specified, the spark channel collapses onto a single point, and there can only be one single spherical kernel point. This scenario is common when no advanced spark plug information is available; such point should be ideally located halfway between the physical electrode locations.
  • If two electrodes are specified, the two 3D locations represent the anode and cathode endpoints. A line-of-sight spark channel will be generated between the two points, with one or more kernel points, equally spaced along it, as specified by the nChannelPts keyword.
    In any cases, the code will check on startup that all specified spark plug locations lay inside the computational domain.

Electrode locations are just specified by the electrode keyword, followed by its x,y,z coordinates separated by spaces; there's no need to specify how many of them should be used. For example, a 2-electrode sparkplug, vertically aligned on the cylinder axis, is:

electrode 0.0 0.0 12.0
electrode 0.0 0.0 11.4

A single-point electrode will just be like:

electrode 1.1 -4.3 0.5

Keyword channelPts describes the number of ignition kernel locations to be placed along the spark plug line. Each parcel represents an ignition location, and experiences locally-accurate thermodynamic and turbulence flow conditions, as long as the underlying mesh resolution allows it. This number is meaningless if only one electrode is specified, as all particles will collapse onto a single location.

note

Channel particles are not allowed to move in the DPIK model: they only change size based on the instantaneous energy release from the spark plug and burn rate.

In the Matlab implementation, electrode locations are stored as either a [3x1] array or a [3x2] sized matrix.

note

There are significant differences as far as what the actual kernel geometry is implemented like. In the original DPIK model, moving Lagrangian parcels and Monte-Carlo methods are used to describe the kernel geometry and kernel volume properties; In FRESCO's implementation, each kernel is represented by a spherical geometry and numerically exact sphere-mesh overlap is employed to interpolate the finite-volume fields.

The initial kernel radius ([cm]) i.e. the spark channel's half-witdth is specified by keyword r0.

Kernel->Flame transition#

Ignition model#

The ignition model represents when and how much energy release from the spark plug to the gas phase takes place. Energy exchange, which leads to kernel growth, is modeled using spark events. The amount of thermal energy transferred to the gas phase is controlled by:

flame.sparkPlug.Qspark = 20; % [W]
flame.sparkPlug.eta_eff = 0.3; % [-]

The thermal source term is computed as:

$$ \dot{Q}=\eta{eff}\dot{Q}{spk}
$$

so the user has control over both the spark plug electrical absorption and the thermal discharge efficiency. This formulation and the 30% discharge average efficiency are attributed to Heywood.

The time-dependent behavior of energy discharge is currently only available as a flat power profile: $$ Q = \int{t{1,ign}}^{t_{2,ign}}{\dot{Q} dt} $$ and lasts unchanged for the whole spark strike duration.

Spark strikes#

The user can specify any number of strike events. Each strike is characterized by a start time and a duration, as in the itapeFlame example:

strike at -14.000000 for 8.000000 ! spark strike 1 of 1

Strike time units should be the same as the simulation time, i.e., crank angle degrees for an engine simulation (input.engine.rpm>0); or seconds otherwise.

In the Matlab structure, each strike is represented by the structure

strike.ca1ign: -1 % Start of ignition event
strike.cadign: -1 % Duration of ignition event

Strikes with negative duration will be ignored.

note

The FRESCO solver will adjust its time-stepping strategy in proximity of the beginning and end of each strike event, to match their timing exactly.
